# تشغيل وكيل التصنيف الحقيقي (n8n) — بصير

هذا الملف `basir_ai_classifier.n8n.json` عبارة عن workflow حقيقي وشغّال في n8n، يستقبل نص بلاغ أو طلب عبر رابط (Webhook) وينادي ذكاء اصطناعي حقيقي (OpenAI) يصنّفه فعليًا إلى بلاغ/طلب + الفئة + نسبة الثقة + حل مقترح — بدل منطق if/else الثابت الموجود داخل ملف بصير نفسه.

## خطوات التشغيل

1. سوّي حساب مجاني على https://n8n.io (أو استخدمي n8n مثبّت عندك محليًا).
2. من داخل n8n: **Workflows → Import from File**، واختاري ملف `basir_ai_classifier.n8n.json`.
3. افتحي عقدة **"نداء الذكاء الاصطناعي الحقيقي (OpenAI)"** واضغطي على حقل الـ Credential، ثم أنشئي Credential جديد من نوع **Header Auth**:
   - Name: `Authorization`
   - Value: `Bearer YOUR_OPENAI_API_KEY` (مفتاحك من https://platform.openai.com/api-keys)
4. فعّلي الـ Workflow (زر Active أعلى يمين الشاشة).
5. انسخي رابط الـ Webhook من عقدة "استقبال البلاغ" (Production URL).
6. جرّبيه مباشرة (مثلاً عبر Postman أو حتى من المتصفح بأداة curl):

```bash
curl -X POST "الرابط_اللي_نسختيه" \
  -H "Content-Type: application/json" \
  -d '{"text": "الطابعة في مكتبي مو شغالة من الصباح، تطلع Offline"}'
```

بترجعلك نتيجة تصنيف حقيقية شكلها كذا:

```json
{
  "family": "incident",
  "category": "أعطال الأجهزة والطابعات",
  "confidence": 88,
  "suggestedSolution": "تحقّقي من توصيل الطابعة بالكهرباء وأنها في وضع التشغيل..."
}
```

## ليه ما ربطناه مباشرة ببصير نفسه؟

اتفقنا إنه بصير (الموقع اللي بتعرضينه) يفضل يشتغل بمنطقه الداخلي المضمون اللي ما يعتمد على إنترنت أو سيرفر خارجي، عشان ما يتعطل لحظة العرض قدام اللجنة/المشرف. هذا الـworkflow دليل منفصل وشغّال فعليًا تقدرين:
- تشغّلينه حي قدام المشرف كدليل إنك استخدمتي n8n وبنيتي وكيل ذكاء اصطناعي حقيقي.
- توثقينه بالتقرير مع لقطة شاشة من n8n وهو يشتغل.

## تحديث: الربط الفعلي (الآمن) داخل بصير نفسه

بما إن الـWorkflow صار شغّال ومنشور فعليًا عندك، أضفنا لبصير زر اختياري باسم **"🤖 جرّب التصنيف عبر وكيل الذكاء الاصطناعي الحقيقي (n8n)"** يظهر تحت حقل "وصف إضافي" في نموذج تقديم البلاغ/الطلب. هذا الزر يستدعي فعليًا رابط الـWebhook حقك (مو محاكاة)، ويعرض نتيجة الذكاء الاصطناعي الحقيقية (family / category / confidence / الحل المقترح) مع زر "استخدم هذا التصنيف" لتطبيقها مباشرة على النموذج.

### خطوة لازم تسويها بنفسك (رابط فقط، بدون أي كلمة مرور أو مفتاح)

1. من داخل بصير، افتحي ملف `basir_prototype.html` بمحرر نصوص (أو اطلبي مني تعديله إذا حبيتي).
2. دوري عن السطر اللي فيه:
   ```js
   let N8N_WEBHOOK_URL = '';
   ```
3. الصقي بين القوسين رابط الإنتاج (Production URL) لعقدة "استقبال البلاغ (Webhook)" اللي نسختيه من n8n، مثال:
   ```js
   let N8N_WEBHOOK_URL = 'https://your-instance.app.n8n.cloud/webhook/basir-classify';
   ```
4. احفظي الملف وافتحيه بالمتصفح — الزر بيصير شغّال فورًا.

### ملاحظة مهمة عن الأمان أثناء العرض

- الزر **اختياري تمامًا** — تقديم البلاغ الأساسي ما يعتمد عليه إطلاقًا.
- إذا فشل الاتصال (شبكة، أو n8n مو شغّال لحظتها) يظهر تنبيه لطيف ويرجع الموقع فورًا لمحركه الداخلي المضمون — ما يصير أي كسر أو تجمّد في العرض.
- مفتاح OpenAI حقك يبقى محفوظ داخل n8n فقط (في الـCredential)، وما يظهر أو يُكتب أبدًا داخل كود بصير أو بالمتصفح.
