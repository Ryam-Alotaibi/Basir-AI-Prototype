# بِصير — نظام دعم فني ذكي | وزارة العدل

**بِصير** نموذج أولي (Prototype) لمنصة دعم فني تعتمد على الذكاء الاصطناعي، مصمّم لتطوير كفاءة نظام "داعم" (القائم على ServiceNow) في وزارة العدل، عبر طبقة استباقية تجمع بين التنبؤ بالأعطال، التصنيف الذكي للتذاكر، والقفزة الذكية للحلول التلقائية.

## 📋 نظرة عامة

يهدف المشروع إلى تعزيز كفاءة واستجابة خدمات الدعم الفني من خلال تقنيات الذكاء الاصطناعي التي تمكّن من:
- توقّع الأعطال قبل وقوعها لضمان استمرارية الخدمات
- تخصيص الموارد بفعالية لتقليل زمن التوقف
- تحسين رضا المستفيدين عبر دعم سريع واستباقي
- خفض التكاليف التشغيلية وتقليل الأعطال المتكررة
- تعزيز الصيانة الاستباقية وتحسين استدامة الأنظمة

المشروع مقدَّم كواجهتين منفصلتين ضمن نفس المنصة:
- **واجهة فني الدعم** — لوحة قيادة كاملة لإدارة التذاكر ومراقبة الوكيل الذكي
- **واجهة الموظف (المستفيد)** — بوابة ذاتية لتقديم البلاغات ومتابعتها

---

## 🎯 الأهداف

1. تحليل الوضع الحالي لنظام "داعم" وتحديد التحديات (الاعتماد الكبير على التدخل البشري في فرز التذاكر، طول مدة الاستجابة للحالات الحرجة، غياب نظام تنبؤي).
2. اقتراح حلول تقنية قائمة على الأتمتة والذكاء الاصطناعي لتحسين سير العمل.
3. تطوير نماذج ذكية للتنبؤ بالأعطال وإدارة التذاكر بكفاءة أعلى.
4. تعزيز تجربة المستخدم ورفع مستوى رضا المستفيدين.
5. تطوير إجراءات لإدارة الحوادث الحرجة تضمن استمرارية الأعمال.

---

## ✨ الميزات

### للفني (Technician)
- **لوحة قيادة تفاعلية**: مؤشرات حوادث حيّة، توزيع التذاكر عبر مراحل دورة الحياة (تسلسل من "قيد العمل" إلى "تم المعالجة")، وأولوية مرتبطة تلقائيًا بالجدول القضائي الفعلي.
- **إدارة تذاكر شاملة**: قائمة انتظار، مساحة عمل لكل تذكرة، اقتراحات ذكاء اصطناعي بنسبة ثقة، وأرشيف قابل للبحث والتصفية.
- **القفزة الذكية (Smart Leap)**: تنفيذ تلقائي للحلول عالية الثقة (≥ 85٪) دون تدخل بشري.
- **متابعة الوكيل الذكي**: سجل شفاف للحالات التي عولجت آليًا بالكامل.
- **التصعيد الذكي**: تصعيد تلقائي للحالات التي تتجاوز حد الثقة الأدنى إلى الدعم البشري.
- **دمج الحوادث**: ربط البلاغات المتكررة لنفس الفئة/الفرع خلال نافذة زمنية في حادثة موحّدة.
- **التنبؤ الاستباقي**: نوافذ خطر متوقعة بناءً على الجدول القضائي، مع توأم رقمي (Digital Twin) لمحاكاة الضغط قبل بدء الأسبوع.
- **لوحة مراقبة حية متعددة الوكلاء** (Multi-Agent Live Telemetry).

### للموظف (Employee)
- **تقديم ذكي**: نموذج بلاغ/طلب موحّد يقترح التصنيف المناسب أثناء الكتابة.
- **تجربة تصنيف بذكاء اصطناعي حقيقي (اختياري)**: تكامل مباشر مع وكيل مبني على n8n + OpenAI لتصنيف حي فعلي (انظر [تعليمات_تشغيل_n8n.md](تعليمات_تشغيل_n8n.md)).
- **تذاكري السابقة وحلولها**، **محادثة فورية مع الوكيل**، و**دليل حلول ذاتية** (Knowledge Base) قابل للبحث.

## 🔭 التطوير المستقبلي

- ربط فعلي بقاعدة بيانات نظام "داعم" (ServiceNow) بدل البيانات الوهمية.
- نموذج تصنيف تذاكر حقيقي مبني على بيانات تاريخية فعلية (NLP) بدل قواعد محاكاة.
- نموذج تنبؤ بالأعطال (Predictive Maintenance) مدرَّب على سجلات الأعطال الحقيقية.
- مصادقة موظفين فعلية عبر نظام شؤون الموظفين (SSO حقيقي) بدل المحاكاة الحالية.
- تطبيق جوال مرافق ودعم الإشعارات الفورية (Push Notifications).
- لوحة تحكم للمشرفين لقياس دقة الوكيل الذكي وتحسينه بمرور الوقت.
